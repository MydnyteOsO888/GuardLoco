 # CarGuard — Flutter Mobile App

Real-time vehicle security system built with Flutter, Firebase FCM, WebRTC, and FastAPI.

---

## Tech Stack

| Layer | Technology | Purpose |
|---|---|---|
| Mobile App | Flutter (Dart) | Cross-platform iOS + Android |
| Push Notifications | Firebase Cloud Messaging (FCM) | Real-time alerts when app is closed |
| Live Video | WebRTC (flutter_webrtc) | Low-latency P2P stream from ESP32-CAM |
| Backend API | FastAPI (Python) | JWT auth, event logging, FCM dispatch |
| Database | PostgreSQL | Events, clips metadata, user data |
| Cloud Storage | AWS S3 / Azure | AES-256 encrypted long-term clip storage |
| State Management | Riverpod | Reactive state across all screens |
| Video Playback | Chewie + video_player | Recorded clip playback |

## Setup

### 1. Prerequisites

```bash
flutter --version   # Requires Flutter 3.19+
dart --version      # Requires Dart 3.0+
```

### 2. Clone and install dependencies

```bash
git clone https://github.com/yourorg/carguard-flutter.git
cd carguard-flutter
flutter pub get
```

### 3. Configure Firebase

```bash
# Install FlutterFire CLI
dart pub global activate flutterfire_cli

# Configure for your Firebase project
flutterfire configure --project=your-firebase-project-id
```

This generates `lib/firebase_options.dart` with your real keys.

### 4. Configure Firebase Cloud Messaging

**Android:** Add `google-services.json` to `android/app/`

**iOS:**
```bash
# Add GoogleService-Info.plist to ios/Runner/
# Enable Push Notifications capability in Xcode
# Enable Background Modes > Remote notifications
```

### 5. Set your API base URL

```bash
# Build with your FastAPI backend URL
flutter run --dart-define=API_BASE_URL=https://your-api.com/api/v1
```

Or create a `.env` file and use `flutter_dotenv`.

### 6. Generate JSON serialization code

```bash
flutter pub run build_runner build --delete-conflicting-outputs
```

This generates `models.g.dart` from the `@JsonSerializable()` annotations.

### 7. Run

```bash
# Debug
flutter run

# Release build
flutter build apk --release                          # Android
flutter build ios --release                          # iOS
```

## ESP32 Integration

The app connects to:

| Endpoint | Method | Description |
|---|---|---|
| `/api/v1/device/status` | GET | Device health, sensors, storage |
| `/api/v1/device/arm` | POST | Arm or disarm system |
| `/api/v1/sensors/stream` | GET (SSE) | Real-time sensor readings stream |
| `/api/v1/events` | GET | Security event log |
| `/api/v1/clips` | GET | Video clip list |
| `/api/v1/clips/:id/stream-url` | GET | Signed S3/local URL for playback |
| `/api/v1/webrtc/offer` | POST | WebRTC SDP signaling |
| `/api/v1/webrtc/ice-candidate` | POST | ICE candidate exchange |
| `/api/v1/settings` | GET/PATCH | Device configuration |

## WebRTC Stream Flow

```
ESP32-CAM  ──── SDP Offer ────▶  FastAPI (signaling)  ◀──── Flutter App
ESP32-CAM  ◀─── SDP Answer ───   FastAPI              ────▶ Flutter App
ESP32-CAM  ──── ICE Candidates ─────────────────────────▶  Flutter App
ESP32-CAM  ◀══════════════ P2P Video Stream (SRTP) ══════  Flutter App
```

```json
    "priority": "high",
    "notification": { "channel_id": "carguard_alerts" }

    "payload": { "aps": { "sound": "default", "badge": 1 } }
  }
}
```

## Security

- **JWT tokens** stored in `flutter_secure_storage` (iOS Keychain / Android Keystore)
- **Token refresh** handled automatically by Dio interceptor
- **WebRTC streams** encrypted with SRTP
- **Cloud video** encrypted with AES-256 at rest (S3 server-side encryption)
- **HTTPS only** — `android:usesCleartextTraffic="false"` enforced
